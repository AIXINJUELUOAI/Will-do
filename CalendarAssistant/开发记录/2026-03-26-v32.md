## 三、关键逻辑摘要

### 3.1 CalendarSync V2
- App → System：单次事件 + 重复事件（RRULE/EXDATE）写入，sync_map 维护
- System → App：重复实例缺失 → 写入 `event_excluded_dates` + `isCancelled`
- 删除清理：本地 master 删除后清理系统重复事件与 sync_map

### 3.2 Room 主存切换
- Room 主存开启后优先读 Room，JSON 仅用于迁移/备份
- 迁移完成 + 7 天无崩溃 → 删除旧 `events.json/archives.json`
- 迁移补齐：新增 JSON recurring 迁移，避免旧重复事件丢失

### 3.3 规则动作/模板/图标
- 状态机默认动作：pending/done/checked_in + 正向/撤销按钮
- 模板渲染：支持 `{title}`/`{field1}` 等占位符，胶囊标题使用模板
- 图标命名：强制使用 `iic_stat_<ruleId>`，无需填写字段

### 3.4 反向同步调度
- WorkManager + Alarm 兜底 + 调试用前台常驻同步（可开关）
- 后台触发仍受系统限制，常驻服务为稳定方案

## 五、风险与注意

- 后台同步在非前台常驻时仍受系统节流影响
- 规则图标需提供 `iic_stat_<ruleId>` 资源，否则回退默认图标
- JSON 清理为 7 天稳定期策略，避免迁移后崩溃导致无法回滚
