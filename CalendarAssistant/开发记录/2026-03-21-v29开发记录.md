## 需求背景

**用户反馈**：课程事件在超级岛/实况胶囊中被错误地显示了"已结束"按钮，但这个按钮没有任何实际用处。

**问题分析**：
- 课程事件有固定的开始和结束时间
- "已结束"按钮用于标记普通日程已完成，但对于课程来说不适用
- 课程通过"调课/停课"功能来管理，而不是简单的完成标记

## 八、本次新增功能（悬浮窗日程范围限制）

### 需求背景

用户希望为悬浮窗添加日程显示范围限制功能，让用户可以控制悬浮窗中显示哪些日程。当前悬浮窗会显示所有符合条件的未来日程，用户希望能够限制为只显示今日或今日+明日的日程，以减少信息过载。

### 功能设计

在偏好设置的悬浮窗区域添加一个三档滑块，允许用户选择：
- **全部日程** (0) - 显示所有未来日程
- **今日日程** (1) - 只显示今天的日程（默认）
- **今日和明日** (2) - 显示今天和明天的日程

UI 设计参考"日程提前提醒"的滑块样式，滑块直接在悬浮窗开启后显示，不需要额外的开关。

#### 4. FloatingScheduleService.kt

**文件位置**：`app/src/main/java/com/antgskds/calendarassistant/service/floating/FloatingScheduleService.kt`

**新增导入**：

import java.time.LocalDate

**新增筛选逻辑**：
```kotlin
setContent {
    val events by repository.events.collectAsState()
    val settings by repository.settings.collectAsState()

    // 根据悬浮窗日程范围设置过滤事件
    val today = LocalDate.now()
    val tomorrow = today.plusDays(1)
    val filteredEvents = when (settings.floatingEventRange) {
        1 -> events.filter { it.startDate == today || it.endDate == today }
        2 -> events.filter {
            it.startDate == today || it.startDate == tomorrow ||
            it.endDate == today || it.endDate == tomorrow
        }
        else -> events // 0 = 全部日程
    }

    // ... 将 filteredEvents 传递给 FloatingScheduleScreen
}
```

### 筛选逻辑说明

| 设置值 | 显示范围 | 筛选条件 |
|--------|----------|----------|
| 0 | 全部日程 | 无过滤 |
| 1 | 今日日程 | startDate 或 endDate 等于今天 |
| 2 | 今日+明日 | startDate 或 endDate 等于今天或明天 |

**边界情况处理**：
- 多日事件（如昨天开始今天结束）→ 包含（endDate == today）
- 跨天事件（今天开始明天结束）→ 包含（startDate == today）
- 只在明天的事件 → 设置为 1 时不显示，设置为 2 时显示
