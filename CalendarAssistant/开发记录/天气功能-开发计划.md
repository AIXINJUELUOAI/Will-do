# 天气功能 — 开发计划

> 日期：2026-04-04
> 状态：规划中

---

## 一、功能概述

为 Will Do 智能日历助手新增天气信息展示，包括主页日期卡片天气显示、悬浮窗天气信息、每日日程提醒附带天气。

**核心设计决策**：
- **不内置天气 API**，用户自行配置 API（与 AI 模型配置模式一致）
- 配置后才显示天气，未配置时 UI 完全不变
- 天气数据本地缓存，按用户设定的频率刷新

---

## 二、用户配置

### 2.1 API 配置项

设置页新增"天气"配置区域，复用 AI 模型配置的 UI 风格：

| 配置项 | 说明 | 示例 |
|--------|------|------|
| API URL | 天气接口地址 | `https://devapi.qweather.com/v7/weather/now` |
| API Key | 密钥 | `your_key_here` |
| 刷新频率 | 1小时 / 3小时 / 6小时（默认3小时） | — |

### 2.2 API 接口规范

用户填入的 API 需满足以下约定（以和风天气为例）：

**请求**：`GET {url}?location={lat},{lon}&key={key}`

**响应**（统一适配格式）：

```json
{
  "temperature": "25",
  "text": "晴",
  "icon": "100",
  "windDir": "东南风",
  "windScale": "3",
  "humidity": "65"
}
```

> 为兼容不同 API 源，内部做一层字段映射，用户选择 API 提供商后自动适配。

### 2.3 支持的 API 提供商（预设映射）

| 提供商 | 说明 |
|--------|------|
| 和风天气 | 国内常用，免费版 1000次/天 |
| OpenWeatherMap | 国际常用，免费版 60次/分钟 |
| 自定义 | 用户自行填写 URL + JSON 字段映射 |

## 三、天气数据管理

### 3.1 数据模型

新建 `data/model/WeatherData.kt`：

```kotlin
@Serializable
data class WeatherData(
    val temperature: String = "",      // "25"
    val text: String = "",             // "晴"
    val icon: String = "",             // 天气图标代码
    val windDir: String = "",          // "东南风"
    val windScale: String = "",        // "3"
    val humidity: String = "",         // "65"
    val updateTime: Long = 0L          // 最后更新时间戳
)
```

### 3.2 缓存策略

- 存储在 `SharedPreferences` 或独立 JSON 文件（量小，不需要 Room）
- 每次打开主页时检查缓存是否过期（对比 `updateTime` 与刷新频率）
- 过期则在后台刷新，未过期直接使用缓存
- 网络失败时继续使用缓存数据，显示"上次更新于 X 分钟前"

### 3.3 定时刷新

- 使用 `WorkManager` 定期拉取天气（PeriodicWorkRequest）
- 间隔由用户配置决定（1h / 3h / 6h）
- 仅在 WiFi 下刷新（可选，默认不限）
- 配置变更时取消旧 Worker，创建新 Worker

## 四、UI 集成

### 4.1 主页日期卡片 — 颜色条区域

**现状**：颜色条（top 20%）为纯色 `primary` 或 `surfaceVariant`

**改造**：颜色条变为渐变色，右侧显示天气信息：

```
┌──────────────────────────────┐
│  渐变色条          ☀️ 25°C   │  ← 颜色条区域（权重 0.2）
├──────────────────────────────┤
│       星期五  初七            │
│                              │
│            4                 │
│                              │
│        2026年4月             │
└──────────────────────────────┘
```

**天气显示内容**：
- 天气图标（晴天☀️ / 多云⛅ / 雨天🌧️ 等，使用内置映射）
- 温度（如"25°C"）
- 点击无跳转（保持简洁）

**颜色条渐变逻辑**：
- 晴天：warm 渐变（橙 → 金）
- 多云/阴：neutral 渐变（灰蓝）
- 雨天：cool 渐变（蓝 → 深蓝）
- 雪：冷色渐变（淡蓝 → 白）
- 未配置天气：保持原纯色不变

### 4.2 悬浮窗 — 事件卡片

**改造**：悬浮窗事件列表顶部新增独立天气卡片，风格与日程卡片一致

```
┌─────────────────────────────┐
│ ☀️  晴  25°C                │  ← 天气卡片（独立卡片）
│     东南风 3级  湿度 65%     │
└─────────────────────────────┘
┌─────────────────────────────┐
│ 🚄 G1234 武汉→北京          │  ← 日程卡片（现有样式）
│ 08:30 - 12:00               │
└─────────────────────────────┘
┌─────────────────────────────┐
│ 📦 菜鸟驿站 1-2-3456        │
│ ...                         │
└─────────────────────────────┘
```

**设计要点**：
- 独立卡片，与日程卡片同圆角、同间距、同配色风格
- 左侧天气图标，右上温度，下方风力+湿度
- 点击可展开查看更多信息（体感温度、能见度等）
- 始终为列表第一项，不随滚动消失（sticky header 或固定位置）

**显示条件**：
- 天气数据存在时显示，不存在则不显示（不占空间）
- 可在设置中关闭悬浮窗天气显示

### 4.3 每日提醒 — 通知内容

**现状**：
```
标题：今日日程提醒
内容：您有 3 个日程：晨会，项目评审，客户拜访
```

**改造**：
```
标题：今日日程提醒 ☀️ 25°C 晴
内容：您有 3 个日程：晨会，项目评审，客户拜访
```

在标题末尾追加天气信息，简洁不突兀。

## 五、定位方案

**优先定位 + 手动兜底**：

1. 用户开启天气功能时，弹出定位权限申请（`ACCESS_COARSE_LOCATION`，城市级精度即可）
2. **用户授权**：自动获取城市信息，无需手动填写
3. **用户拒绝**：回退到手动输入城市名或城市 ID（如 "武汉" 或 "101200101"）

**权限流程**：
```
开启天气开关 → 请求 ACCESS_COARSE_LOCATION
  ├── 授权 → 自动获取城市 → 请求天气 API
  └── 拒绝 → 显示城市输入框 → 用户手动填写 → 请求天气 API
```

**AndroidManifest 新增**：
```xml
<uses-permission android:name="android.permission.ACCESS_COARSE_LOCATION" />
```

## 九、实施顺序

### 第一阶段：数据层
1. `WeatherData` 数据模型
2. `WeatherApiAdapter` — API 请求 + 响应映射
3. `WeatherRepository` — 缓存管理 + 定时刷新
4. `MySettings` 新增字段
5. `SettingsViewModel` 支持更新

### 第二阶段：设置页 UI（独立侧边栏入口）
1. `SettingsDestination` 新增 `Weather` 枚举值
2. 侧边栏添加"天气"入口（图标 + 标题，与"模型配置"平级）
3. 新建 `WeatherSettingsPage`（API URL / Key / 城市 / 提供商 / 刷新频率 / 悬浮窗开关）
4. "测试连接"按钮验证配置是否正确

### 第三阶段：主页日期卡片
1. `WeatherIconMapper` + `WeatherColorMapper`
2. `MainViewModel` 暴露 `weatherData`
3. 日期卡片颜色条改为渐变 + 天气图标温度

### 第四阶段：悬浮窗
1. 传入天气数据到悬浮窗 Service
2. `FloatingScheduleScreen` 顶部新增独立天气卡片（与日程卡片风格一致）
3. 天气卡片支持点击展开详情
4. 设置项控制显隐

### 第五阶段：每日提醒
1. `DailySummaryReceiver` 拉取天气缓存
2. 标题追加天气信息
