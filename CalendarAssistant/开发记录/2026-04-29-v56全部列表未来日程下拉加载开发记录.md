# 2026-04-29 开发记录 (v56)：全部列表未来日程下拉加载

## 一、目标

优化“全部列表页”的未来日程展示范围。

当前全部列表页按“未来 -> 过去”的方向排列，越远的未来越靠上，因此用户希望通过顶部下拉来加载更远未来的日程。

目标行为：

```text
初始显示：历史全部 + 未来 7 天
第一次下拉：未来范围扩展到 22 天
第二次下拉：未来范围扩展到 37 天
后续每次下拉：继续 +15 天
```

本功能只扩大 UI 展示窗口，不改数据库，不重新拉取远程数据。

---

## 二、实现内容

### 1) ViewModel 展示窗口状态

文件：

- `app/src/main/java/com/antgskds/calendarassistant/ui/viewmodel/MainViewModel.kt`

改动：

- 在 `MainUiState` 增加：

```kotlin
val allEventsFutureDays: Int = 7
val allEventsFutureLimit: LocalDate = LocalDate.now().plusDays(7)
```

- 在 `MainViewModel` 中新增内部状态：

```kotlin
private val _allEventsFutureDays = MutableStateFlow(INITIAL_ALL_EVENTS_FUTURE_DAYS)
```

- `uiState` 的 `combine(...)` 纳入 `_allEventsFutureDays`。
- `futureLimit` 从固定 `today.plusDays(7)` 改为：

```kotlin
val futureLimit = today.plusDays(allEventsFutureDays.toLong())
```

- 新增方法：

```kotlin
fun loadMoreFutureAllEvents() {
    _allEventsFutureDays.value += ALL_EVENTS_LOAD_MORE_DAYS
    _revealedItemKey.value = null
}
```

- 常量：

```kotlin
private const val INITIAL_ALL_EVENTS_FUTURE_DAYS = 7
private const val ALL_EVENTS_LOAD_MORE_DAYS = 15
```

### 2) 全部列表页下拉加载

文件：

- `app/src/main/java/com/antgskds/calendarassistant/ui/page_display/AllEventsPage.kt`

改动：

- 使用 Material3 `PullToRefreshBox` 包裹全部列表页。
- 下拉触发时调用：

```kotlin
viewModel.loadMoreFutureAllEvents()
```

- 本地使用 `isLoadingMoreFuture` 控制加载态。
- 当 `uiState.allEventsFutureDays` 变化后，关闭加载态。

## 三、UI 调整

### 1) 初版提示

最初在列表顶部加入了提示：

```text
已显示至 x月x日 · 下拉加载未来 15 天
```

但用户反馈放在列表外观里不好看。

### 2) 调整为下拉指示器内提示

后续改为自定义下拉指示器：

```text
[加载圆环] 已显示到 xx，下拉加载到 xx
```

触发加载后显示：

```text
[加载圆环] 正在加载到 xx
```

实现方式：

- 自定义 `AllEventsFutureLoadIndicator(...)`。
- 使用 `PullToRefreshState.distanceFraction` 控制圆环进度和指示器位置。
- 使用 `Surface + Row + CircularProgressIndicator + Text` 展示提示。
- 去掉列表里的固定顶部提示 item。
- 空状态文案恢复简洁，不再显示“下拉加载更远未来”。

### 3) 编译报错修复

后续用户编译发现：

```text
Unresolved reference 'RoundedCornerShape'
```

原因是自定义指示器使用了 `RoundedCornerShape`，但缺少 import。

已在 `AllEventsPage.kt` 补充：

```kotlin
import androidx.compose.foundation.shape.RoundedCornerShape
```

## 四、当前行为

当前设计下：

- 全部列表页初始显示到未来 7 天。
- 下拉后未来范围增加 15 天。
- 因列表排序是“未来 -> 过去”，新加载的更远未来日程会出现在列表更靠上的位置。
- 提示文案不占据列表内容区域，而是跟随下拉加载图标显示。
