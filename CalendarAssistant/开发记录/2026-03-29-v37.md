## 二、适配过程与踩坑记录

### 2.1 全屏导航栏适配

**问题**：启用 `enableEdgeToEdge()` 后，内容被系统导航栏遮挡，底部留白不一致。

**解决方案**：
- 所有 Scaffold 添加 `contentWindowInsets = WindowInsets(0)`，阻止空 `bottomBar` 自动消耗 insets
- 浮动底栏使用 `.navigationBarsPadding()` 自动适配手势导航和三键导航
- LazyColumn 通过 `contentPadding = PaddingValues(bottom = floatingBarOffset)` 为底栏预留空间
- 可滚动设置页面（主题/实验室/关于/捐赠）底部添加 `windowInsetsBottomHeight(WindowInsets.navigationBars)` Spacer

### 2.2 小米小窗小白条适配（多次尝试）

**问题**：小窗模式下，底部拖拽手柄周围出现一圈纯白色边框（与 App 主题色不一致）。

**尝试过的方案（均失败）**：

| 方案 | 结果 |
|------|------|
| `WindowInsets.navigationBars.asPaddingValues().calculateBottomPadding().coerceAtLeast(16.dp)` | 小窗下 insets 为 0，无效 |
| `WindowInsets.systemBars` 替代 `navigationBars` | 小窗下同样返回 0 |
| `isInMultiWindowMode` + 手动 28dp padding | 内容上移但暴露更多白色窗口背景，白边更明显 |
| 32dp coerceAtLeast | 同上 |
| 根容器加 `.padding(bottom = freeformBottomPadding)` | 白边依旧 |
| `themes.xml` 加 `windowOptOutEdgeToEdgeEnforcement=true` | 全屏有效但与 `enableEdgeToEdge()` 冲突，小窗依旧白边 |
| `window.setBackgroundDrawable(ColorDrawable(主题背景色))` | SideEffect 中设置但无效 |

**最终成功的方案（三管齐下）**：

1. **最外层 Box 加主题背景色**：`Modifier.fillMaxSize().background(MaterialTheme.colorScheme.background)` — 确保背景色铺满整个窗口（包括小白条区域），只有交互控件避让
2. **显式关闭导航栏对比度保护**：在 `onCreate` 中 `enableEdgeToEdge()` 之后添加 `window.isNavigationBarContrastEnforced = false` — 防止系统为保护小白条可见性自动加白色底色
3. **移除 `windowOptOutEdgeToEdgeEnforcement`**：与手动 `enableEdgeToEdge()` 冲突，移除后由 App 完全控制 edge-to-edge 行为

**核心教训**（来自 Gemini 分析）：
> 小窗白边不是因为没有避开小白条，而是因为避开后留下的空隙没被背景色填满。背景色应铺满全窗口，只有交互控件才需要避让。

### 2.3 关键风险：DensityConfigManager 不可修改

`attachBaseContext` 中使用 `createConfigurationContext` 进行 UI 密度缩放（0.75-0.85x）。曾尝试改为 `applyOverrideConfiguration` 导致整个 APP 放大且无法调整，已回退。

## 四、最终适配架构

```
MainActivity.setContent
└── CalendarAssistantTheme
    ├── SideEffect: statusBarColor/navigationBarColor=透明, isNavigationBarContrastEnforced=false
    └── Box(fillMaxSize + background=主题背景色)  ← 背景
        └── NavHost
            ├── HomeScreen
            │   ├── PushSlideLayout
            │   │   └── HomePage
            │   │       ├── Scaffold(contentWindowInsets = WindowInsets(0))
            │   │       └── LazyColumn(contentPadding = bottom + floatingBarOffset)
            │   └── IntegratedFloatingBar(navigationBarsPadding)  ← 只有控件避让
            └── SettingsDetailScreen
                └── Scaffold(contentWindowInsets = WindowInsets(0))
                    └── 各设置页面
                        └── 底部 Spacer(windowInsetsBottomHeight)
```

**适配原则**：
- **背景铺满**：最外层容器用主题背景色填满整个窗口
- **控件避让**：只有浮动底栏等交互控件使用 `navigationBarsPadding()`
- **列表留白**：LazyColumn 通过 `contentPadding` 预留空间，内容可滚动到 padding 区域
- **全屏与小窗一致**：不区分模式，统一用同一套逻辑处理
