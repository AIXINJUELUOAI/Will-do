# 2026-03-28 开发记录 (v36版本)

## 一、本次实现范围

1. **原生实况通知修复（Live Activity 上岛）**：对比参考项目 Hyper-pick-up-code 发现前台服务通知无法被 Android 提升为实况通知，分离前台服务与胶囊通知后成功上岛
2. **去掉反射调用**：`NativeCapsuleProvider` 中 `setRequestPromotedOngoing` 和 `setShortCriticalText` 改为直接 API 调用（API 35+ / 36+）
3. **移除 CapsuleService 前台服务**：将 CapsuleService 的通知管理逻辑合并到 CapsuleStateManager，删除前台服务和"胶囊服务运行中"通知
4. **识别状态通知合并**：OCR 进度和结果共用同一个通知 ID（88886），分析完成时直接更新而非取消再创建
5. **OCR 与日程胶囊共存**：修复 OCR 胶囊活跃时会顶掉所有日程胶囊的 BUG，提取 `computeScheduleCapsules()` 方法实现共存
6. **Description 标签中文化**：`normalizeDescriptionHeader()` 将英文 ruleId 标签统一替换为中文名
7. **NativeCapsuleProvider 精简**：移除 `setCategory`、`setGroup`、`setForegroundServiceBehavior`、`createPromotionExtras` 等干扰实况通知提升的配置
8. **通知 ID 集合线程安全修复**：`CapsuleStateManager.activeNotifIds` 从 `mutableSetOf` 改为 `ConcurrentHashMap.newKeySet()`，修复多协程并发读写导致的偶发性 `ConcurrentModificationException` 崩溃（归档事件时触发）
9. **长按音量+行为重构**：改为完全拦截 + 短按补偿，长按动作可配置为【无操作/识屏/悬浮窗】，并支持悬浮窗关闭时无操作
10. **设置页新增“操作”板块**：独立展示长按音量+开关与动作滑块，原悬浮日程描述调整为悬浮窗功能说明
11. **底部浮层间距优化**：识别中提醒与搜索条的底部间距调整，避免与底栏相接

## 三、关键逻辑摘要

### 3.1 实况通知上岛原理

对比参考项目发现，Android 不允许前台服务通知被提升为 Live Activity。修复前 CapsuleService 通过 `startForeground()` 将胶囊通知作为前台服务通知发送，导致无法上岛。

修复后：
- 胶囊通知全部通过 `notificationManager.notify()` 发送（普通通知）
- NativeCapsuleProvider 在 API 35+ 通过 `Bundle("android.requestPromotedOngoing", true)` 请求提升
- API 36+ 通过 `setShortCriticalText()` 在状态栏胶囊中显示简短文本
- 去掉 `setCategory(CATEGORY_EVENT)`、`setGroup()`、`setForegroundServiceBehavior()` 等干扰项

### 3.2 CapsuleService 移除

原 CapsuleService 职责全部合并到 CapsuleStateManager：
- `startServiceWakeup()` → `startNotificationManager()`：监听 uiState 变化，直接发布/更新/取消通知
- `upsertCapsule()` → `updateCapsules()`：`notificationManager.notify()` + stale 清理
- `cleanupStaleCapsules()` → `cleanupStaleNotifications()`：每 3 秒检查过期通知
- 前台服务依赖移除，依赖用户开启自启动保持后台运行

### 3.3 OCR 与日程胶囊共存

原 `computeCapsuleState()` 中 OCR 胶囊活跃时直接 `return`，丢弃所有日程胶囊。修复后提取 `computeScheduleCapsules()` 方法，OCR 分支调用后合并：`listOf(ocrItem) + scheduleCapsules`。

### 3.4 依赖说明

移除前台服务后，APP 依赖用户开启自启动（国产 ROM 基本默认开启给常用 APP）。已有 `ongoing` 通知在进程被杀后仍保留在通知栏，只是不会自动更新。

### 3.5 长按音量+动作分流

长按音量+统一在无障碍层拦截：短按在抬手时补偿音量并显示系统音量条，长按根据设置动作触发识屏或悬浮窗。若用户选择“悬浮窗”但悬浮窗总开关未启用，则直接无操作处理。
