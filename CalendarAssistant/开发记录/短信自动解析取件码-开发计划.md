# 短信自动解析取件码 — 开发计划

> 日期：2026-03-30
> 状态：待开发

---

## 一、需求概述

监听手机短信，自动识别快递取件码（含取餐码），解析后直接入库为日程事件，并发送通知告知用户。

**核心原则**：
- 复用现有 `MyEvent` 表，不加新字段，当作 AI 识别返回的取件码事件处理
- 自动入库 + 通知告知，无需用户确认
- 默认关闭，用户需手动开启并授予短信权限

## 三、文件变更清单

### 新建文件（2 个）

#### 1. `core/sms/SmsParser.kt` — 短信解析器
- `object SmsParser` 单例，纯 Kotlin，无 Android 依赖
- 仅处理 `106` 开头的短信号码
- 核心方法：
  - `parse(sender, body): ParsedSmsPickup?` — 主入口，返回解析结果或 null
  - `extractPickupCode(body): String?` — 提取取件码
  - `extractLocation(body): String` — 提取地址（优先快递柜地址，其次通用地址）
  - `resolveCompany(sender, body): String` — 识别快递公司

- 数据模型：
```kotlin
data class ParsedSmsPickup(
    val pickupCode: String,    // 取件码，如 "1-2-3456"
    val platform: String,      // 平台，如 "菜鸟驿站"
    val companyName: String,   // 快递公司，如 "顺丰速运"
    val location: String,      // 取件地址，如 "北门快递柜"
    val tag: String            // EventTags.PICKUP 或 EventTags.FOOD
)
```

#### 2. `service/receiver/SmsReceiver.kt` — 短信广播接收器
- 继承 `BroadcastReceiver`，监听 `SMS_RECEIVED`
- 使用 `goAsync()` + `CoroutineScope(Dispatchers.IO)` 模式
- 流程：
  1. 检查 `isSmsMonitoringEnabled` 开关
  2. 调用 `SmsParser.parse()` 解析
  3. 去重：`repository.events.value` 中是否已有相同取件码
  4. 构建 `MyEvent`，描述使用微格式 `【取件】code|company|location`
  5. 调用 `repository.addEvent()` 入库
  6. 发送通知

### 修改文件（4 个）

#### 3. `data/model/MySettings.kt`
- 在 `isPickupAggregationEnabled` 后添加：`val isSmsMonitoringEnabled: Boolean = false`

#### 4. `ui/viewmodel/SettingsViewModel.kt`
- `updatePreference()` 参数列表添加：`smsMonitoring: Boolean? = null`
- 方法体添加：`if (smsMonitoring != null) current = current.copy(isSmsMonitoringEnabled = smsMonitoring)`

#### 5. `AndroidManifest.xml`
- 添加权限：`RECEIVE_SMS`、`READ_SMS`
- 注册 SmsReceiver，`exported="true"`，priority 999

#### 6. `ui/page_display/settings/PreferenceSettingsPage.kt`
- 添加 `smsPermissionLauncher` 权限请求回调
- 在"通知"卡片中取件码聚合开关后添加"短信自动解析取件码"开关
- 开启时检查权限，未授权则请求

## 五、注意事项

1. **权限**：`RECEIVE_SMS` 是危险权限，需运行时请求（minSdk 26 全部需要）
2. **去重**：同一条短信可能因网络重试收到两次，需按取件码去重
3. **兼容性**：`kotlinx.serialization` 对新增字段有默认值兼容，不影响已有用户数据
4. **性能**：正则匹配 + 一次 DB 写入，`goAsync()` 的 10 秒限制足够
5. **安全**：仅处理 106 开头的号码，避免读取私人短信
6. **默认关闭**：`isSmsMonitoringEnabled = false`，用户必须主动开启
