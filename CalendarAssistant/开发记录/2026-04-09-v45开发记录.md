## 五、便签编辑能力重构过程

### 1. 初始目标

本轮原计划包括：

- 便签从 Dialog 改为独立页面
- 支持 Markdown 输入与展示
- 支持 todo 语法
- 支持从便签内容一键 AI 分析生成日程

### 2. 中途尝试与问题

先后尝试了以下几条路线：

- 结构化 `todo` + 弹窗编辑
- 全屏编辑页 + `Markwon` 原位编辑
- `MarkwonEditor + EditHandler` 增强编辑态渲染
- “当前行源码 / 其他行渲染”的混合方案

在实测中暴露出大量问题：

- 光标与焦点状态不稳定
- 点击区域与链接事件冲突
- 列表、待办、链接在编辑态中频繁出现边界问题
- 每输入一个字就整篇重算，导致状态错乱与内容丢失

### 3. 最终方向调整

在多轮问题暴露后，决定放弃“单一大文本框里同时完成源码编辑与所见即所得”的方案，转为更清晰的块级编辑器方向：

- 用户输入与最终存储都保持为原始 Markdown 文本
- 内部通过块模型识别标题、段落、引用、待办、列表、表格等结构
- 当前块进入编辑态时显示源码
- 非当前块显示渲染态
- 编辑期间以缓存为主，退出编辑页时统一保存

这条路线比前面的 `Span` 补丁方案更适合手机端，也更符合便签作为“补充能力”的产品定位

## 七、悬浮窗便签模式正式接入

### 目标

解决便签与日程混排逻辑不清的问题，将悬浮窗改为“日程 / 便签”两种模式，通过底部左端按钮切换，并记住上次状态

### 本轮实现

- 悬浮窗支持 `FloatingInputMode.SCHEDULE / NOTE` 模式切换
- 模式与底部输入区左端按钮联动
- 记住上次停留的模式，下次打开悬浮窗自动恢复
- `日程模式` 仅展示天气 + 日程流
- `便签模式` 仅展示天气 + 便签卡列表

### 便签卡逻辑收口

便签卡最终采用 3 态设计：

1. 默认态
   - 白色圆角矩形卡片
   - 展示标题与最后修改日期
   - 右上角仅在存在 todo 时显示进度胶囊（如 `2/5` / `已完成`）

2. 展开态
   - 点卡片主体展开/收起
   - 优先展示 todo，再展示正文摘要
   - 总内容不超过 6 行
   - todo 允许直接勾选

3. 编辑态
   - 左滑露出“编辑”按钮
   - 点击编辑后自动展开并进入源码编辑
   - 底部按钮：取消 / 保存

### 删除逻辑

- 右滑触发删除
- 删除后底部展示“已删除便签，是否撤销操作”提示
- 撤销时限为 5 秒

### UX 收益

- 便签不再和日程混在一条时间轴里，语义更清晰
- 输入模式与主内容区统一，减少上下文割裂感
- 悬浮窗中的便签能力从“详情卡拼接”升级为完整的模式化交互

## 九、天气图标全面替换

### 背景

- 原天气图标使用 Material Icons（`ImageVector`），风格单一，与天气场景匹配度不高
- 用户手动导入了 15 个自定义矢量图标，覆盖常见天气类型

### 替换方案

新增 15 个 drawable 资源文件：

| 图标文件 | 对应天气 |
|---------|---------|
| `ic_weather_sunny` | 晴（白天） |
| `ic_weather_clear_night` | 晴（夜间） |
| `ic_weather_partly_cloudy` | 多云（白天） |
| `ic_weather_partly_cloudy_night` | 多云（夜间） |
| `ic_weather_overcast` | 阴 |
| `ic_weather_rain` | 雨 |
| `ic_weather_rain_heavy` | 暴雨 |
| `ic_weather_thunderstorm` | 雷电 |
| `ic_weather_snow` | 雪 |
| `ic_weather_snow_heavy` | 暴雪 |
| `ic_weather_sleet` | 雨夹雪 |
| `ic_weather_fog` | 雾 |
| `ic_weather_haze` | 霾 |
| `ic_weather_wind` | 大风/沙尘/台风 |
| `ic_weather_hail` | 冰雹 |

### 代码改动

- `WeatherIconMapper.kt`：函数签名从 `fun icon(data: WeatherData): ImageVector` 改为 `fun iconRes(data: WeatherData): Int`，返回 drawable 资源 ID
- 映射逻辑基于 QWeather 天气文本和 icon code，优先级为：雷 > 雹 > 雨夹雪 > 暴雪 > 雪 > 暴雨 > 雨 > 台风 > 沙尘 > 霾 > 雾 > 阴 > 多云 > 晴
- 夜间图标通过 `isNightIcon()` 辅助函数区分
- 调用处（`HomePage`、`FloatingScheduleScreen`、`WeatherSettingsPage`）统一改为 `Icon(painter = painterResource(WeatherIconMapper.iconRes(...)))`

## 十、天气同步 Worker 错误处理修复

### 问题

- `WeatherSyncWorker.doWork()` 的 catch 块返回 `Result.success()`
- 网络异常或解析失败时 Worker 被视为成功完成，不会重试，用户静默丢失天气数据

### 修复

- catch 块改为返回 `Result.retry()`，让 WorkManager 在满足约束条件后自动重试

## 十一、便签删除对话框修复

### 问题

1. **高度过高**：FloatingActionCard 底部使用 `bottomSafePadding + 16.dp`（其中 bottomSafePadding = 112.dp + extraBottomPadding），导致对话框位置偏高
2. **TopBar 未变暗**：遮罩层位于 NotePage 内部（Scaffold content 区域），无法覆盖到 TopBar

- 将 `pendingDeleteNote` 状态从 NotePage 提升到 HomePage
- 遮罩层和 FloatingActionCard 移至 HomePage 的 Scaffold 外层 Box（与"正在识别"对话框同级）
- 底部间距改为 `floatingBarOffset + 16.dp`，与"正在识别"对话框保持一致

### 涉及文件

- `ui/page_display/NotePage.kt`：新增 `onPendingDeleteChange` 参数，移除内部遮罩和对话框
- `ui/page_display/HomePage.kt`：承接删除状态和对话框 UI

## 十二、便签页视觉二次收口

### 背景

- 在第一轮便签页改版后，列表虽然已经从双列瀑布流改成单列，但整体仍偏“立体卡片”
- 与 `全部日程` 页当前的扁平排版语言相比，存在明显割裂
- 同时列表页直接展示较多 Markdown 结构内容，扫读时显得嘈杂

### 本轮调整

#### 1. 列表整体改为纯扁平风格

- 去掉概览大卡片、厚底色、明显阴影与大面积浅色容器
- 顶部仅保留一行轻量统计文本
- 每条便签改为平面信息行，通过细分隔线组织层级

#### 2. 列表项展示策略收口

- 每条仅展示：`标题 + 一行摘要/任务统计 + 更新时间`
- 若便签包含 todo，则优先显示任务统计（如“3 项待办，2 项已完成”）
- 不再在便签列表中逐条展开任务，也不再完整渲染 Markdown 视觉结构

#### 3. Markdown 摘要提纯

- 对正文摘要做轻量清洗，去掉：
  - 标题符号 `#`
  - 引用符号 `>`
  - 任务框 `- [ ]`
  - 列表/有序列表前缀
  - 行内代码包裹符号
  - 粗体、斜体、删除线等格式标记
- 目的不是在列表页“展示 Markdown”，而是快速提炼可扫读的信息摘要

#### 4. 编辑页动作视觉收敛

- 将便签编辑页的大圆形主色 AI FAB 收敛为更小、更克制的圆角按钮
- 按钮容器色切换到 `secondaryContainer` 体系，避免抢占正文注意力

### 效果

- 便签页与 `全部日程` 页的视觉语言重新拉齐
- 列表页从“内容展示墙”收口为“信息索引页”
- 编辑页与列表页的主次关系更清晰：列表负责识别，编辑页负责查看与修改完整内容
