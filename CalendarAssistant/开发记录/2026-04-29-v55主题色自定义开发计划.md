# 2026-04-29 开发计划 (v55)：主题色自定义

## 一、目标

为 App 主题色系统增加“自定义颜色”能力。用户可以在现有预设主题色之外，自行选择 RGB 颜色，并由该颜色生成整套 Material3 主题色板。

本计划只针对软件主题色，不涉及日程标签色、事件颜色或胶囊事件颜色。

---

## 二、当前结构

当前主题色链路：

```text
MySettings.themeColorScheme
  -> ThemeColorScheme.fromName(...)
  -> CalendarAssistantTheme(...)
  -> ThemeColorGenerator.generateColorScheme(seedColor, darkTheme, schemeName)
  -> MaterialTheme.colorScheme
```

主要文件：

- `app/src/main/java/com/antgskds/calendarassistant/data/model/MySettings.kt`
  - 当前保存 `themeMode`、`themeColorScheme`。
- `app/src/main/java/com/antgskds/calendarassistant/ui/theme/ThemeColorScheme.kt`
  - 当前保存固定主题色枚举。
- `app/src/main/java/com/antgskds/calendarassistant/ui/theme/Theme.kt`
  - 当前选择动态色或固定 seed 色。
- `app/src/main/java/com/antgskds/calendarassistant/ui/theme/ThemeColorGenerator.kt`
  - 基于 seed color 生成 Material3 色板。
- `app/src/main/java/com/antgskds/calendarassistant/ui/page_display/settings/ThemeSettingsPage.kt`
  - 当前展示主题模式和固定主题色网格。
- `app/src/main/java/com/antgskds/calendarassistant/ui/viewmodel/SettingsViewModel.kt`
  - 当前提供主题模式和主题配色更新方法。
- `app/src/main/java/com/antgskds/calendarassistant/service/floating/EdgeBarService.kt`
  - 侧边栏单独读取主题色，需要同步支持自定义色。

## 三、设计方案

### 1) 设置字段

在 `MySettings` 增加字段：

```kotlin
val customThemeColor: Int = 0xFF6750A4.toInt()
```

说明：

- 使用 ARGB Int 保存颜色。
- 默认值使用当前经典紫，兼容旧配置。
- 不需要数据库迁移；设置使用序列化默认值即可兼容。

### 2) 主题枚举

在 `ThemeColorScheme` 增加：

```kotlin
CUSTOM("自定义", Color(0xFF6750A4))
```

说明：

- `CUSTOM` 只作为模式标记。
- 实际 seed color 读取 `settings.customThemeColor`。

### 3) 主题生成

调整 `CalendarAssistantTheme`：

- 增加参数 `customThemeColor: Color`。
- 当 `themeColorScheme == CUSTOM` 时，使用 `customThemeColor` 作为 seed。
- 仍然走 `ThemeColorGenerator.generateColorScheme(...)`。

这样可以保证：

- 浅色/深色模式自动适配。
- `primary`、`onPrimary`、`primaryContainer`、`surface` 等颜色保持 Material3 对比关系。
- 不直接硬改 `primary`，避免文字可读性问题。

### 4) 设置页入口

在 `ThemeSettingsPage` 的主题颜色区域加入“自定义”入口。

建议位置：

- 作为主题色网格中的最后一个卡片；或
- 放在主题色网格下方的独立按钮。

推荐文案：

```text
自定义颜色
使用 RGB 调整应用主题色
```

点击后打开 `ModalBottomSheet`。

### 5) 自定义颜色 Sheet

Sheet 内容：

```text
标题：自定义主题色

[大色块预览]
RGB: 103, 80, 164
HEX: #6750A4

R  [Slider 0..255]  103
G  [Slider 0..255]   80
B  [Slider 0..255]  164

[主题预览区]
- 一个按钮
- 一个卡片
- 一段主色文字

[取消] [应用]
```

交互规则：

- 拖动滑块只更新 Sheet 内本地状态。
- 点击“应用”后才写入设置。
- 点击“取消”或关闭 Sheet 不保存。
- 不开放 Alpha，统一使用不透明色，避免透明主题色导致文字/胶囊/侧边栏显示异常。

### 6) ViewModel 接口

在 `SettingsViewModel` 增加方法：

```kotlin
fun applyCustomThemeColor(color: Int) {
    viewModelScope.launch {
        settingsOperationApi.updateSettings(
            settings.value.copy(
                themeColorScheme = ThemeColorScheme.CUSTOM.name,
                customThemeColor = color
            )
        )
    }
}
```

必要时也可拆分为：

- `updateCustomThemeColor(color: Int)`
- `selectCustomThemeColor()`

但第一版建议一次性应用，减少状态复杂度。

## 四、需要同步适配的地方

### 1) MainActivity

当前主题切换监听：

```kotlin
LaunchedEffect(settings.themeMode, settings.themeColorScheme)
```

后续需要加入：

```kotlin
settings.customThemeColor
```

否则自定义颜色更新后可能不会触发 Activity 重建或主题刷新。

### 2) FloatingScheduleService

悬浮窗内部也使用 `CalendarAssistantTheme`，需要传入自定义主题色。

### 3) EdgeBarService

侧边栏当前通过 `settings.themeColorScheme` 解析主色。

需要增加：

```text
themeColorScheme == CUSTOM -> settings.customThemeColor
```

否则自定义主题色和侧边栏颜色会不一致。

---

## 五、边界与限制

- 自定义色仅影响 App 主题色，不影响：
  - 日程事件颜色
  - 标签颜色
  - 系统日历颜色
  - 胶囊事件本身颜色
- `DEFAULT` 仍表示跟随系统动态色。
- `CUSTOM` 表示用户自选 seed 色。
- 如果用户选择过亮或过暗的颜色，Material3 生成器会尽量保证文字对比，但设置页仍应提供预览。
- 不建议第一版增加历史颜色、收藏色、吸管等复杂功能。

---

## 六、验证清单

1. 选择预设主题色仍正常生效。
2. 选择“跟随系统”仍正常走系统动态色。
3. 设置自定义 RGB 后，App 主题立即或重启后生效。
4. 自定义主题色在浅色模式下可读。
5. 自定义主题色在深色模式下可读。
6. 自定义主题色重启后保留。
7. 悬浮窗主题跟随自定义色。
8. 侧边栏颜色跟随自定义色。
9. 不影响日程颜色、胶囊事件颜色、系统日历同步颜色。

---

## 七、建议实施顺序

1. 增加 `MySettings.customThemeColor`。
2. 增加 `ThemeColorScheme.CUSTOM`。
3. 调整 `CalendarAssistantTheme` 支持自定义 seed color。
4. 调整 `MainActivity` 和悬浮窗主题传参。
5. 调整 `EdgeBarService` 主题色解析。
6. 在 `SettingsViewModel` 增加应用自定义主题色方法。
7. 在 `ThemeSettingsPage` 增加自定义入口和 Sheet。
8. 编译并按验证清单测试。
