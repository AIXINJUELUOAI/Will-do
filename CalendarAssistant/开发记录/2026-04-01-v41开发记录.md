## 二、短信自动解析取件码

### 背景

用户收到快递取件短信后需要手动记录，希望能自动识别并创建日程事件。

### 技术方案

复用现有 AI 识屏管线，正则解析短信后生成 `CalendarEventData`（与 AI 输出格式完全一致），下游直接走现有的入库 + 通知流程。

### 架构设计

```
SMS_RECEIVED → SmsReceiver → SmsAnalysis.parse() → CalendarEventData → eventDataToMyEvent() → Repository + Notification
```

核心思路：只有 `SmsAnalysis` 和 `SmsReceiver` 两个文件，`SmsAnalysis` 负责正则解析，`SmsReceiver` 负责广播接收和入库。

### 解析能力

- **取件码提取**：支持"取件码"、"提货号"、"凭"、"快递"等关键词，匹配 `[A-Za-z0-9-]` 格式
- **地址提取**：支持"X号柜"、"X号快递柜"及通用地址关键词匹配
- **平台识别**：菜鸟驿站、丰巢、京东、美团、饿了么、盒马等
- **快递公司识别**：顺丰、中通、圆通、韵达、申通、极兔、邮政、京东、德邦等
- **外卖分类**：美团、饿了么、盒马、叮咚买菜、肯德基等自动标记为取餐类型
- **忽略过滤**：验证码、还款、消费、余额、转账等金融类短信自动跳过

### 106 号码过滤

仅在 `SmsReceiver` 中过滤 106 开头号码，`SmsAnalysis` 本身不做此限制，保持解析器通用性。

### 输出格式对齐

与 AI 识屏输出格式完全一致：

- **标题**：`📦 菜鸟驿站 1-2-3456` / `🍔 美团 A05`
- **描述微格式**：`【取件】1-2-3456|顺丰速运|北门快递柜` / `【取餐】A05|美团|`
- **标签**：`EventTags.PICKUP` / `EventTags.FOOD`

## 三、实验室页 UI 统一

### 问题

实验室页的提示文字样式与偏好设置页不一致。

### 偏好设置页样式体系

```kotlin
val cardTitleStyle = bodyLarge + Medium + onSurface
val cardSubtitleStyle = bodyMedium + onSurfaceVariant(0.6f)
// 警告提示：cardSubtitleStyle + error 色 + Warning 图标
```

### 修复内容

- **短信警告提示**：`bodySmall` → `bodyMedium`，`onSurfaceVariant` → `error`，padding 对齐为 `horizontal = 16.dp, vertical = 8.dp`
- **规则编辑器提示**：`bodySmall` → `bodyMedium`，颜色改为 `onSurfaceVariant.copy(alpha = 0.6f)`（无警告图标，不用红色）
