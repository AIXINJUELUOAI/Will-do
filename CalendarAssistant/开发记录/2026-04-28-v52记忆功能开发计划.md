# 2026-04-28 v52 开发记录

## 主题

语音输入、便签迁移与 Markdown 文件夹同步规划。

## 背景

项目后续希望从“日程识别助手”逐步扩展为“用户主动授权的个人信息处理台”，形成“识别 + 入库 + 整理 + 分析 + 结论 + 搜索”的长期方向。在进入完整记忆库之前，先补齐低门槛记录和旧笔记迁移能力，让用户可以无缝把想法、清单和现有便签带入 Will do。

第三方 App 不具备小布记忆一类系统级私有接口、后台常驻唤醒和跨应用深度读取能力，因此近期规划不追求复刻系统级体验，而是优先做好用户主动触发、可追溯、可导出、可迁移的记录闭环。

## 产品目标

- 语音输入降低记录门槛，用户说一句话即可生成便签或日程。
- 便签导入导出降低迁移成本，支持用户从旧笔记软件搬迁内容。
- Markdown 文件夹同步提供开放格式，兼容外部编辑器和第三方同步工具。
- 现有便签保持轻量编辑体验，同时为后续记忆库沉淀结构化资料。
- 不依赖系统便签、小布记忆或其他厂商私有 API。

## 阶段一：语音输入 MVP

### 范围

- 新增 `RECORD_AUDIO` 权限，并在用户点击语音入口时按需申请。
- 优先接入 Android 系统 `SpeechRecognizer` 做应用内语音转文字。
- 暂不保存原始音频，只保存转写文本，降低隐私和存储压力。

### 入口

- 便签编辑页：语音追加到当前正文。
- 首页快捷入口：录音后选择保存为便签、识别为日程或自动判断。
- 悬浮窗输入框：复用现有手动输入通道，支持语音快速输入。

### 处理链路

- 保存为便签：使用 `createNoteEvent()` 创建 `Event(tag = NOTE)`。
- 识别日程：继续走 `RecognitionCenter.parseUserText(..., sourceType = VOICE_CAPTURE, ingestRequested = true)`。
- 自动判断：先由规则或 AI 判断是便签、日程还是混合内容，再分发到对应链路。

## 阶段二：便签专用导入导出

### 范围

- 在数据管理页新增“便签数据”卡片，避免和“日程数据”混淆。
- 保留现有日程备份能力；便签专用备份只处理 `EventTags.NOTE`。

### 导出格式

- 原生 JSON：完整保留标题、正文、颜色、完成状态、更新时间等元数据。
- Markdown：一条便签一个 `.md` 文件，适合迁移、外部编辑和长期同步。

### 导入格式

- `.json`：导入本应用便签备份。
- `.md`：按 Markdown 文件生成便签。
- `.txt`：按纯文本生成便签。

### 去重与冲突

- 优先使用导入数据中的 `willdo_id` 或内部 ID。
- 没有 ID 时，用“标题 + 正文 hash”识别重复。
- 冲突默认保留两份，不静默覆盖用户数据。

## 阶段三：Markdown 文件夹同步

### 范围

- 先做本地/文档树目录同步，不直接实现账号制云同步。
- 用户通过系统文件选择器选择同步目录。
- 用户可以自行配合坚果云、OneDrive、Syncthing、系统文件提供器等工具完成跨设备同步。

### 文件格式

每条便签对应一个 Markdown 文件，文件头保存必要元数据：

```markdown
---
willdo_id: note_xxx
title: 标题
updated_at: 1710000000
color: 0
state: pending
---

正文内容
```

### 同步策略

- 本地改、外部未改：写出到文件。
- 外部改、本地未改：更新本地便签。
- 两边都改：生成冲突副本，不自动覆盖。
- 删除策略第一版先保守处理：外部删除不立即删除本地，可标记为待确认。

## 阶段四：体验整合

- 便签页逐步升级为“笔记/便签”入口，但暂不直接改成完整记忆库。
- 新用户引导包含“导入旧笔记”“选择同步文件夹”“试试语音记录”。
- 语音结果页保持轻量，核心按钮为“存便签”“识别日程”“重新录音”。
- 后续再接入分享入口、截图 OCR 入便签、AI 总结和记忆库搜索。

## 架构边界

- 语音输入只作为内容来源，不绕过现有 Center/API/Store 主链。
- 日程类内容继续走识别与入库中心，不在 UI/Service 中直接写业务流程。
- 便签仍可暂时复用 `Event(tag = NOTE)`，但不要把完整记忆库事实源直接塞进 `Event`。
- 系统 Calendar Provider 不承载便签同步。
- 不尝试读取小布记忆、系统便签或其他厂商私有数据。

## 推荐实现顺序

1. 实现语音输入基础组件与权限处理。
2. 接入便签编辑页和悬浮窗输入框。
3. 增加便签专用导入导出。
4. 实现 Markdown 文件夹同步。
5. 再讨论分享入口、截图 OCR 入便签和完整记忆库模型。
