## 二、随口记数据层与中心层

### 2.1 新增随口记表

新增 `QuickMemoEntity`，用于承载轻量文本/语音随口记：

- `type`: `TEXT` / `VOICE`
- `bodyText`
- `audioPath`
- `audioDurationMs`
- `transcriptionStatus`
- `analysisStatus`
- `createdAt`
- `updatedAt`
- `todoState`: `NONE` / `ACTIVE` / `COMPLETED`
- `todoPendingUntil`
- `todoCompletedAt`

本轮保留 `todoPendingUntil` 字段以兼容早期方案和后续扩展，但当前交互不使用 5 秒待办撤销态。

### 2.2 新增随口记日程候选表

新增 `QuickMemoSuggestionEntity`，用于保存随口记 AI 分析得到的日程候选：

- `type`: 当前只做 `SCHEDULE`
- `status`: `PENDING` / `CREATED` / `DISMISSED` / `FAILED`
- `quickMemoId`
- `candidateJson`
- `eventId`
- `createdAt`
- `updatedAt`

候选 JSON 通过 `QuickMemoSuggestionCodec` 保存/恢复 `RecognitionDraft`。

### 2.3 Room 升级

`EventsDatabase` 从 version 7 升级到 version 8：

- 新增 `quick_memos` 表。
- 新增 `quick_memo_suggestions` 表。
- 普通便签 `notes` 表不动，降低迁移风险。

### 2.4 新增 QuickMemoCenter

新增 `QuickMemoCenter` 作为随口记业务中心，负责：

- 文本随口记创建。
- 语音随口记创建。
- 正文更新。
- 待办完成/撤回。
- 删除随口记并清理音频文件。
- 语音转写异步处理。
- 转写成功后触发多事件文本分析。
- 日程候选生成和通知触发。

## 四、悬浮窗文本随口记

### 4.1 便签模式改为随口记模式

悬浮窗原“便签”输入模式改为“随口记”输入模式：

- 输入框占位文案改为“记一条随口记...”。
- 文本提交后创建 `QuickMemo(TEXT)`。
- 不再写入普通便签 `notes` 表。
- 悬浮窗列表展示随口记卡片。

### 4.2 文本随口记规则

文本随口记保持极简流程：

- 输入文本。
- 点击发送。
- 直接保存为普通随口记。

文本随口记本轮不做创建时的“普通/待办”二次确认。

## 五、语音随口记与音量键录音

### 5.1 音量+长按设置

`volumeUpLongPressAction` 新增动作值：

```kotlin
3 = 语音输入
```

行为规则：

- 设置为“语音输入”：长按音量+开始录音，松手结束。
- 设置为“悬浮窗”：第一次长按打开悬浮窗；悬浮窗已显示时再次长按进入语音输入。
- 短按音量+仍补偿系统音量增加。

### 5.2 悬浮窗最小录音态

语音输入复用同一个悬浮窗容器，不再创建第二个窗口：

- 长按开始后，底部输入框原地切换为浮动波形线条。
- 录音中显示“松手保存”。
- 松手后输入框进入确认态。
- 录音太短时提示“录音太短”，不保存。

### 5.3 语音确认态

松手后先保存临时音频，再进入 icon-only 确认态：

- 待办 icon：保存为待办随口记。
- 普通保存 icon：保存为普通随口记。
- 5 秒未选择时默认保存为普通随口记。
- 关闭悬浮窗时若处于确认态，也默认保存为普通随口记，避免丢失录音。

### 5.4 录音和播放

新增音频基础能力：

- `QuickMemoAudioRecorder`: 使用 `MediaRecorder` 录音。
- 录音保存为 `.m4a/AAC` 到私有目录：`files/quick_memos/audio/`。
- 小于 500ms 的录音不保存。
- `AudioPlaybackCenter`: 使用 `MediaPlayer` 播放。
- 全局保证同一时间只播放一个音频。
- 删除语音随口记时同步清理音频文件。

## 六、语音卡片与待办交互

### 6.1 语音卡片

语音随口记卡片支持：

- 折叠态显示波形和播放按钮。
- 点击波形展开正文/转写状态。
- 点击播放按钮播放/暂停音频。
- 展开后显示转写文本或状态。

展开态状态包括：

- `转写中`
- `转写失败，可重试`
- `仅音频`
- 转写成功后的正文文本

转写失败时提供“重试转写”按钮。

### 6.2 待办交互

待办随口记支持：

- 左滑展开“完成/撤回”和“编辑”按钮。
- 长左滑直接完成/撤回。
- 卡片使用浅暖色背景、黄色左侧竖线、小待办标识和 checkbox 状态。

本轮不再提供“普通随口记左滑转待办”。

## 七、ASR 本地模型架构

### 7.1 sherpa-onnx 架构接入

新增依赖：

```kotlin
implementation("com.bihe0832.android:lib-sherpa-onnx:6.25.21")
```

新增 ASR 抽象与实现：

- `SpeechTranscriber`
- `NoopSpeechTranscriber`
- `SherpaParaformerTranscriber`
- `QuickMemoAudioDecoder`
- `QuickMemoAsrModelStore`

### 7.2 不内置模型、不提供下载链接

根据当前产品决策，本轮只保留 ASR 架构和本地模型加载能力：

- APK 不内置大模型。
- 软件内不提供下载链接。
- 软件内不提供下载按钮。
- 用户后续自行下载模型文件，再通过设置页导入。

当前本地模型目录：

```text
files/quick_memos/asr/sherpa-onnx-paraformer-zh-small-2024-03-09/
```

当前加载器期望导入：

```text
model.int8.onnx
tokens.txt
```

也兼容 `model.onnx` 作为非 int8 模型文件名。

### 7.3 设置页模型导入入口

偏好设置页新增“语音转写模型”项：

- 显示当前模型就绪状态。
- 支持导入 `.onnx` 模型文件。
- 支持导入 `tokens.txt`。
- 不展示外部下载地址。

### 7.4 当前模型类型限制

当前实现使用 `OfflineParaformerModelConfig`。

如后续改用 SenseVoice，需要扩展 ASR 模型类型配置和加载器，目前尚未实现 SenseVoice 分支。

## 八、AI 分析与候选通知

### 8.1 文本多事件分析入口

新增文本多事件分析入口：

- `RecognitionProcessor.analyzeTextEvents()`
- `RecognitionCenter.analyzeTextEvents()`

语音转写成功后走该入口，复用 OCR/图片识别的多候选解析逻辑。

### 8.2 AI 只生成日程候选

随口记语音转写后：

- 不自动入库日程。
- 不自动生成待办。
- 只生成日程候选。
- 候选写入 `quick_memo_suggestions`。

### 8.3 候选通知

识别到随口记日程候选后逐条通知：

```text
识别到日程 | title
startTime - endTime
```

通知 action：

```text
创建事件
```

点击后：

- 从 `candidateJson` 恢复 `RecognitionDraft`。
- 创建正式日程。
- 更新 suggestion 状态为 `CREATED`。
- 写入 `eventId`。
- 取消对应候选通知。

### 8.4 已直接入库结果通知逐条化

OCR/截图这类已直接入库结果改为逐条通知：

```text
新增事件 | title
startTime - endTime
```

多条通知使用独立 notificationId，并加入同一 notification group。
