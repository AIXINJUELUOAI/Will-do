# 2026-03-22 开发记录 (v30版本)

## 需求背景

**用户反馈**：
1. 聚合胶囊图标显示错误，当第一个取件码是取餐类型时，聚合胶囊会错误显示取餐图标
2. 魅族实况胶囊 UI 设计过时，希望与最新的设计风格保持一致

**问题分析**：
- 聚合胶囊的图标判断逻辑直接复用了第一个事件的 description 字段
- 魅族胶囊使用白色背景 + 深色文字，与实况胶囊场景不符
- 图标资源分散在 ic_capsule_* 和 ic_stat_* 两个系列，维护成本高

---

## 一、本次实现范围

1. **修复聚合胶囊图标**：始终使用包裹图标，不受单个取件码类型影响
2. **重构魅族胶囊 UI**：透明背景、图标左移、统一文字样式
3. **图标资源统一**：全部改用 ic_stat_* 系列
4. **清理未使用资源**：删除 7 个未使用的图标文件

## 三、修改详情

### 3.1 聚合胶囊图标修复

**文件**：`app/src/main/java/com/antgskds/calendarassistant/service/capsule/IconUtils.kt`

**问题**：聚合胶囊直接使用第一个取件码的 description 判断是否为取餐，导致图标显示混乱。

**修改前**：
```kotlin
CapsuleService.TYPE_PICKUP, CapsuleService.TYPE_PICKUP_EXPIRED -> {
    if (isFoodPickup(capsule.description)) R.drawable.ic_stat_food else R.drawable.ic_stat_package
}
```

**修改后**：
```kotlin
fun getSmallIconForCapsule(capsule: CapsuleUiState.Active.CapsuleItem): Int {
    // 聚合取件码胶囊始终使用包裹图标
    if (capsule.id == com.antgskds.calendarassistant.core.capsule.CapsuleStateManager.AGGREGATE_PICKUP_ID) {
        return R.drawable.ic_stat_package
    }
    // ...
}
```

### 3.2 魅族胶囊图标资源统一

**文件**：`app/src/main/java/com/antgskds/calendarassistant/service/capsule/provider/FlymeCapsuleProvider.kt`

**问题**：魅族胶囊使用独立的 ic_capsule_* 系列图标，与原生胶囊不一致。

**修改**：`resolveFlymeIcon()` 方法全部改用 ic_stat_* 系列

| 场景 | 修改前 | 修改后 |
|------|--------|--------|
| 取件码 | ic_capsule_pickup | ic_stat_package |
| 火车票 | ic_capsule_train | ic_stat_train |
| 网约车 | ic_capsule_taxi | ic_stat_car |
| 课程 | ic_capsule_course | ic_stat_course |
| 普通事件 | ic_capsule_event | ic_stat_event |

---

### 3.3 魅族胶囊 UI 重设计

**文件**：
- `app/src/main/res/layout/notification_live_flyme.xml`
- `app/src/main/res/layout/notification_live_network_speed.xml`

**改动对比**：

| 项目 | 修改前 | 修改后 |
|------|--------|--------|
| 背景色 | `@android:color/white` | `@android:color/transparent` |
| 布局方向 | 文字左 + 图标右 | 图标左 + 文字中 |
| 图标大小 | 48dp（独立） | 40dp（直接显示，无背板）|
| 主标题 | 22sp bold, #1A1A1A | 18sp medium, #F2F2F2 |
| 副标题 | 14sp, #757575 | 12sp, #99FFFFFF |
| 字间距 | 无 | 0.03 |
| padding | 16dp | 12dp (上下) / 14-20dp (左右) |

**新布局结构**：
```
┌────────────────────────────────────┐
│ [40dp图标]  主标题 (18sp, 白)      │
│             副标题 (12sp, 半透明白) │
└────────────────────────────────────┘
```
