# 特殊日程类型隔离方案（生日/纪念日/倒数日）

## Context

开启日历同步后，系统日历中的生日、纪念日等事件会作为重复日程同步到 APP，混在普通日程列表中造成信息干扰。这些事件本质上是"里程碑日期"而非"待办日程"，用户需要的信息也不同（年龄、第几年、还有几天），需要从普通日程中隔离出来单独展示。

---

## 步骤 3：事件识别与映射

**文件**: `core/calendar/CalendarEventMapper.kt`

识别逻辑（在 `mapSystemEventToMyEvent` 中）：

```
检测条件：customAppPackage == "com.android.calendar" && allDay && isRecurring
  ├─ systemEventType == 2  → tag = BIRTHDAY
  ├─ systemEventType == 1  → tag = ANNIVERSARY
  ├─ systemEventType == 3  → tag = COUNTDOWN
  └─ systemEventType == null 且 FREQ=YEARLY → tag = BIRTHDAY（兜底）
```

识别到特殊事件时：
- `eventType` 设为 `SPECIAL_DATE`
- `tag` 设为对应的标签
- `originalDate` 从标题中解析年份：`"张三 (1990)"` → `LocalDate.of(1990, month, day)`，解析失败则为 null

## 步骤 4：ViewModel 过滤

**文件**: `ui/viewmodel/MainViewModel.kt`

`MainUiState` 新增：
```kotlin
val specialDateEvents: List<MyEvent> = emptyList()
```

`uiState` 的 `combine` 中：
- 从 `events` 分离出 `specialDateEvents = events.filter { it.eventType == EventType.SPECIAL_DATE }`
- `todayNormal`、`todayMerged`、`tomorrowMerged` 计算时使用 `normalEvents`（排除 SPECIAL_DATE）

## 步骤 5：主页展示

**文件**: `ui/page_display/HomePage.kt`

在日期卡片、权限警告之后、"今日安排"之前，插入"特别日期"区域：

```
[日期卡片]
[权限警告（如有）]
[特别日期] ← 新增区域
  - 生日卡片: "张三 · 26岁生日 · 5月20日"
  - 纪念日卡片: "结婚 · 第3年 · 10月1日"
[今日安排]
[明日安排]
```

卡片内容计算：
- **生日**: `ChronoUnit.YEARS.between(originalDate, today)` → "N岁生日"
- **纪念日**: `ChronoUnit.YEARS.between(originalDate, today)` → "第N+1年"
- **倒数日**: `ChronoUnit.DAYS.between(today, startDate)` → "还有N天" / "已过N天"

只在当天或即将到来（7天内）的特殊日期事件才显示，避免列表过长。

图标：`Icons.Default.Cake`（生日）、`Icons.Default.Favorite`（纪念日）、`Icons.Default.Event`（其他）

## 步骤 6：全部日程页过滤

**文件**: `ui/page_display/AllEventsPage.kt`

`filteredEvents` 计算中添加过滤：
```kotlin
.filter { it.eventType != EventType.SPECIAL_DATE }
```

---

## 步骤 7：保护特殊事件

**文件**: `data/repository/AppRepository.kt`

- `autoArchiveExpiredEvents()`: 跳过 `eventType == SPECIAL_DATE` 的事件
- 特殊事件不支持归档/删除操作（来自系统日历，应通过系统日历管理）

---

## 验证方式

1. 在系统日历中创建联系人生日事件
2. 开启日历同步，确认事件被识别为 SPECIAL_DATE 类型
3. 主页"特别日期"区域正确显示，包含年龄/年数计算
4. 生日/纪念日不出现在普通日程列表中
5. 不被自动归档
6. 原有日程功能不受影响
