# 2026-03-28 开发记录 (v35版本)

## 一、本次实现范围

1. **实况通知图标选择**：规则编辑弹窗新增图标选择器，用户可从预设图标库中为每条规则选择胶囊通知图标，支持"使用默认"恢复
2. **新增 4 个内置规则**：flight（航班）、food（取餐）、ticket（取票）、sender（寄件），总内置规则扩展到 8 个（日程类 4 + 取件类 4）
3. **AI 提示词更新**：提示词头加入全部 8 个场景的 Title 模板（含 emoji）与 description 格式，规则补丁补齐 food/ticket/sender，schedule/pickup prompt 的 tag 列表同步更新
4. **模板系统无括号化**：默认模板改为纯中文字段名（如"取件码""登机口 登机"），渲染时系统自动根据规则的 aiPrompt 解析字段名并转换为 `{fieldN}` / `{title}` 占位符
5. **航班模板对齐列车**：航班 pending 从 `{field1}` 改为 `登机口 登机`，terminal 从 `{title}` 改为 `座位号`
6. **规则编辑弹窗 UI 统一**：从 AlertDialog 改为 Dialog + Card 样式，与新建日程/新建课程保持一致（RoundedCornerShape 28dp、85% 宽、最大 670dp 高）
7. **图标选择器 UI 统一**：从 Dialog 改为 AlertDialog，与时间/日期选择器风格一致（surface 背景、28dp 圆角、tonalElevation 6dp）
8. **深浅模式自适应 UI**：底栏与日期卡片根据 `surface.luminance()` 动态切换颜色层级，兼容 App 内手动切换主题
9. **UI 细节优化**：移除底栏多余 24dp padding（Edge to Edge）、日期卡片圆角加大至 16dp、移除"启用"和"应用于日程"开关
10. **新增 drawable 资源**：ic_stat_flight.xml、ic_stat_ticket.xml、ic_stat_sender.xml（替换为用户提供的 SVG 图标）、iic_stat_general.xml、iic_stat_train.xml、iic_stat_taxi.xml、iic_stat_pickup.xml
11. **Description 标签中文化**：`normalizeDescriptionHeader()` 将所有事件的 description 标签从英文 ruleId（如 `【pickup】`）统一替换为中文名（如 `【取件】`）

## 四、性能优化与 UI 修复

### 4.1 列表滑动卡顿优化
- **删除调试 stackTrace**：`AppRepository.updateEvents()` 中每次事件更新都调用 `Thread.currentThread().stackTrace.joinToString()` 抓取完整调用栈，严重阻塞主线程，已删除
- **精确过期定时器**：`MainViewModel` 中 10 秒轮询改为精确计算最近事件过期时间点，只在事件实际过期时触发 UI 刷新；无未过期事件时保底 60 秒检查一次，大幅减少无效重组
- **合并重复 isEventExpired**：`AllEventsPage` 排序中 `compareBy` 3 个选择器改为 `sortedWith` 单次比较器，`DateCalculator.isEventExpired()` 从每个事件调 2 次减为 1 次
- **归档页避免重复加载**：`ArchivesPage` 的 `LaunchedEffect` 加入 `archivedEvents.isEmpty()` 判断，已有数据时跳过加载

### 4.2 实验室页卡顿优化
- **页面加载跳过 refresh**：`LaboratoryPage.loadRules()` 不再调用 `RuleRegistry.refresh()`（6+ DB 查询 + N+1 transition 查询 + getIdentifier 反射），仅读取 `ruleDao.getAll()`
- **图标资源缓存**：`RuleIconPickerDialog` 中 12 个预设图标的 `getIdentifier()` + `toBitmap()` 用 `remember` 缓存；`RuleIconPreview` 的 `getIdentifier()` 同样缓存
- **N+1 查询消除**：`RuleRegistry.refresh()` 中逐 rule 查 transition 改为一次 `getAll()` 查询在内存分组，`EventTransitionDao` 新增 `getAll()` 方法
- **规则开关切换精简**：toggle `isEnabled` 只更新 DB 并刷新列表，不触发 `RuleRegistry.refresh()`

### 4.3 其他优化
- **PreferenceSettingsPage 事件订阅移除**：该页面为了一处重复提醒检查订阅了全量 events Flow，任何事件变化都触发重组，改为 `repository.events.value` 按需读取
- **实验室页卡片圆角统一**：Card 默认圆角（12dp）统一改为 `RoundedCornerShape(16.dp)`，与偏好设置页保持一致

### 4.4 修复编译错误
- `RuleIconPicker.kt`：补充 `import androidx.compose.runtime.remember`，`Icon` 调用改为显式 `painter =` 参数消除 `BitmapPainter` 重载歧义

## 五、关键逻辑摘要

### 4.1 图标解析优先级链
用户自定义 (RuleRegistry.customCapsuleIconCache) → 默认 (iconCache / iic_stat_{ruleId}) → 硬编码 when 回退

### 4.2 模板无括号化
- 默认模板存储纯中文："取件码"、"登机口 登机"
- RuleDisplayTemplateResolver.convertFriendlyPlaceholders() 渲染前解析规则的 aiPrompt，构建 `fieldName → {fieldN}` 映射并替换
- 同时兼容带花括号 `{field1}` 和不带花括号的写法，向后兼容

### 4.3 图标选择器数据流
- 选择图标 → 直接更新 DB 的 iconSourceJson → RuleRegistry.refresh() → 列表即时刷新
- RuleIconPreview 支持 iconResName 参数，编辑时选图标无需等 DB 保存即可预览

### 4.4 深浅模式自适应
- 使用 `MaterialTheme.colorScheme.surface.luminance() < 0.5f` 判断深色模式
- 兼容 App 内手动切换主题（不依赖 isSystemInDarkTheme）
- 深色模式色阶：surface（背景）→ surfaceContainerHigh（底栏）→ surfaceContainerLow（卡片）

### 4.5 Description 标签中文化
- `normalizeDescriptionHeader()` 将 description 中的英文 ruleId 标签替换为中文：general→日程、train→列车、taxi→用车、flight→航班、pickup→取件、food→取餐、ticket→取票、sender→寄件
- 前端展示通过 `RuleMatchingEngine.resolvePayload()` 已兼容中英文标签，此处修改确保原始文本也统一为中文
