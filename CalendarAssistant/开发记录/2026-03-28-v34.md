# 2026-03-28 开发记录 (v34版本)

## 一、本次实现范围

1. **规则引擎工具聚合重构**：将分散在多个 Util 类中的解析逻辑（`TransportUtils`、`PickupUtils`）聚合到 `EventPresenter` 统一渲染入口，所有 UI 组件通过 `EventPresenter.present()` 获取渲染数据
2. **Bug 修复 — 主页两个地点**：定位并修复了 APK 签名不同导致 Android Studio 无法覆盖安装旧版本的问题，用户需卸载后重新安装
3. **Bug 修复 — 动态标题越界**：主页/全部列表页标题改为使用原始 `event.title`，动态标题仅用于悬浮窗/胶囊/通知
4. **状态解析去 description 耦合**：`isCheckedIn` 和 `isCompleted` 状态完全依赖 DB/内存字段，不再从 description 尾部解析 `(已检票)`、`(已用车)` 标记
5. **悬浮窗过期标题回退**：悬浮窗中过期事件显示原始标题而非动态标题
6. **移除重复日程同步开关**：数据模型重构后重复日程已成为一等公民，移除"同步重复日程 (Beta)"独立开关，日历同步统一处理所有事件
7. **移除同步后台常驻开关**：删除实验室中的调试用前台服务（`CalendarSyncKeepAliveService`/`CalendarSyncKeepAlivePrefs`）
8. **AI 提示词标签中文化**：AI 生成的 description 标签从英文 ruleId（`【pickup】`/`【train】`/`【taxi】`/`【general】`）改为中文名（`【取件】`/`【列车】`/`【用车】`/`【日程】`）

## 三、关键逻辑摘要

### 3.1 EventPresenter 统一渲染
- `present(context, event)` 返回 `EventRenderModel`，包含 title/subtitle/detail/statusLabel/action 等全部展示数据
- 内部根据 `resolveRuleId()` 分发到 `resolveTrainDisplay` / `resolveTaxiDisplay` / `resolvePickupDisplay` / `resolveGeneralDisplay`
- 动态标题由 `RuleDisplayTemplateResolver.renderTitle()` 从 DB 模板渲染，支持 `{title}`、`{location}`、`{field1~5}` 等占位符

### 3.2 显示位置与标题策略
- **主页/全部列表页**（SwipeableEventItem）：使用 `event.title`（原始标题）
- **悬浮窗**（FloatingScheduleScreen）：未过期用 `model.title`（动态标题），过期用 `event.title`
- **胶囊/通知**（CapsuleMessageComposer）：始终使用动态标题

### 3.3 状态字段独立性
- `event.isCheckedIn`：仅通过 `checkInTransport()` / `undoCheckInTransport()` 修改 DB 字段
- `event.isCompleted`：仅通过 `completeScheduleEvent()` / `undoCompleteEvent()` 修改 DB 字段
- description 不再追加 `(已检票)`、`(已用车)` 后缀，保持原始内容干净
- `parseTransport()` 不再 `removeSuffix`，直接用原始 description 解析 payload

### 3.4 Seeder 强制同步
- `BUILT_IN_RULE_IDS` 集合包含 general/train/taxi/pickup 四个内置规则
- 内置规则在每次启动时通过 `INSERT OR REPLACE` 强制覆盖 DB 中的 `displayTemplate` 为最新默认值
- 自定义规则仅补缺失，不覆盖用户修改

### 3.5 重复日程同步统一
- 移除 `isRecurringCalendarSyncEnabled` 开关，`allowRecurringSync` 固定为 `true`
- 日历同步时自动处理重复日程，不再需要用户手动开启
- 胶囊/通知/提醒的过滤条件统一为 `!event.isRecurringParent`

### 3.6 AI 提示词标签中文化
- `AiPrompts.kt` 中所有提示词模板的 description 格式改为中文标签
- `RuleMatchingEngine.normalizeRuleId()` 已兼容中英文标签（如 `"取件"/"pickup"` 均映射到 `RULE_PICKUP`）
- 新增 `"日程"` → `RULE_GENERAL` 映射

### 3.7 APK 覆盖安装问题
- **根因**：设备上安装的是 release 签名版本，Android Studio 的 debug build 使用 debug 签名，签名不同无法覆盖
- **现象**：代码编译成功但设备上运行的仍是旧版本代码
- **解决**：先卸载设备上的旧版本，再从 Android Studio Run
