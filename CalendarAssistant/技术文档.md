## 1. 项目概述

Will do（智能日历助手）是一个基于 Android 的智能日程管理应用，核心目标是把「识别 -> 入库 -> 同步 -> 提醒/通知展示」做成稳定闭环。

当前工程已经完成从旧架构向新架构收口：

- UI 层统一通过 ViewModel 调用 Center
- 业务层通过 Operation/Query API 对外暴露能力
- 数据层由 `StoreDispatcher` + `StoreRootNode` 组织 Node 子模块
- 主存储使用 Room，保留 JSON 迁移/兼容能力

## 3. 兼容性与系统能力

### 3.1 Android 版本兼容

- Android 8.0（API 26）起可运行核心功能
- 截图识别依赖 `AccessibilityService.takeScreenshot`，需 Android 11（API 30）及以上

### 3.2 实况通知/胶囊说明（重要）

- 原生实况通知（Live Activity）按 Android 16（API 36）能力说明
- 在 Android 16 上，通知可结合 promoted ongoing 与 `setShortCriticalText` 提升胶囊展示效果
- Android 16 以下不按原生 Live Activity 能力描述，应用会回落到普通通知或厂商适配路径
- Flyme 通过 `RemoteViews + extras` 适配胶囊展示
- HyperOS 超级岛通过 Xposed 模块在 SystemUI 侧分发展示

## 4. 当前架构（v57 收口后）

### 4.1 分层结构

```text
UI/ViewModel
  -> Center (Schedule/Sync/Reminder/Runtime/...)
    -> Core API (operation/query contracts)
      -> StoreDispatcher
        -> StoreRootNode
          -> Store*Node 子模块
            -> Room / JSON 迁移兼容 / CalendarSync
              -> System Calendar
```

### 4.2 关键设计要点

- 架构主线为 `Center -> API -> Node`
- `StoreDispatcher` 直接实现核心 `OperationApi/QueryApi`
- `StoreRootNode` 负责数据编排与模块协作（同步、迁移、归档、提醒）
- 顶层通过 `App.kt` 进行依赖装配，不在 UI 层直接触达数据实现

## 5. 数据模型与存储策略

### 5.1 主存与迁移

- 主存储：Room
- 兼容迁移：保留 JSON 读取与迁移流程（事件、归档、重复事件）
- 启动期会进行数据清洗/迁移检查，并更新运行时 StateFlow

### 5.2 课程与重复日程统一

- 课程主模型并入 `MyEvent(tag=course)`
- 课程按重复日程语义处理（父事件/实例/排除实例）
- 课程读写不再依赖独立课程运行时流，而是从事件投影

## 6. 核心能力模块

### 6.1 识别与导入

- 文本识别、截图识别、图片识别、多模态识别
- AI 解析后落地为结构化事件，进入统一写入链路
- Prompt 支持本地版本与云端更新

### 6.2 日历同步

- App 与系统日历双向同步
- 支持单次与重复事件
- 通过映射与去重避免重复写入/回环

### 6.3 胶囊/通知

- 胶囊类型包含：日程、取件聚合、网速、OCR 进度与结果
- `CapsuleStateManager` 统一计算状态并发布通知
- 已移除旧 `CapsuleService` 前台服务路径

### 6.4 运行时能力

- 悬浮窗与 EdgeBar 侧边栏
- 无障碍权限恢复（Shizuku/Root 相关）
- 异常日志与轻量 ANR 监控
